# Scan-JSON agent-usability audit, 2026-06-10

[design](design.md) §2 names scan-JSON evidence richness as **the** lever for consumer 1
(LLM agents calling nose): the output "should carry *why* two units are equivalent,
*what* differs, exact locations, and the behavior contract, so the agent can decide and
act without re-deriving the analysis." This audit measured, for the first time, whether
that loop actually closes ([#216]). The checked-in data artifact is
[bench/labels/scanjson_agent_audit_2026_06_10.json](../bench/labels/scanjson_agent_audit_2026_06_10.json).

## Protocol

Deterministic sample: the top default-surface family from each of 14 repos (2 per
corpus language), plus 4 review-surface exact fragments. For each family the
consumer-1 task was attempted using **only the scan-JSON object** — decide
worthy / not-worthy and sketch the refactor — with the decision recorded *before*
opening any source file, then graded against the source.

## Score

**14 of 18 decided correctly from the JSON alone.** Paths plus the structural stats
(members/files/languages, shared lines, params, scope) carry more than expected:
test-data fixtures, locale tables, module-format twins, and engine/app parallel models
were all correctly rejected from path shape alone. The four failures are the gap list:

| family | JSON-only call | truth | what was missing |
|---|---|---|---|
| cmark `scanners.c` (6×27-line prod blocks) | worthy | **generated** — line 1 says `Generated by re2c`; it still topped the default surface | a generated-content signal |
| arrow `locales.py` (33 shared lines, 1 param) | worthy | locale **data tables** (identical CJK month arrays) | difference evidence |
| antlr4 Java↔JS (languages=2, `shared_lines: 0`, `mean_score: 1.0`) | undecidable | real cross-language Type-4 in parallel runtime ports | an equivalence witness — nothing in the family says *why* it merged |
| alacritty `vi_mode.rs` (`scope: "prod"`) | worthy | `#[test]` fns in an inline `mod tests` — **the scope tag is wrong** | Rust inline-test-module awareness in `is_test_loc` |

The four review fragments fared better on the *why* — `fragment_kind` and
`reason_code` are exactly the witness default families lack — but an agent still
cannot *act* on one: the enclosing unit name is absent and the extraction contract
the proof established (parameters, live-outs) is not serialized.

## Ranked gaps (each proven by a blocked decision above)

1. **No equivalence witness on default families.** There is no `channel` field, no
   proof provenance, no score breakdown — `shared_lines: 0` with `mean_score: 1.0` is
   uninterpretable. Fragments already carry `reason_code`; whole-unit families carry
   nothing. *Fix: emit the producing channel per family, plus proof provenance for
   `semantic` and score components for `near`.*
2. **No difference evidence.** `params: N` counts the varying spots but not where or
   what they are; `--show diff` exists for humans only. *Fix: serialize per-member
   varying-spot spans (the abstraction-witness layer already computes typed holes for
   a subset).*
3. **Generated-content markers neither surfaced nor honored.** `Generated by re2c` /
   curl's generated-sample banner sit in the file head; the generated-file *path*
   heuristic misses them, so they not only lack a flag — they top the default surface.
   *Fix: cheap file-head scan → per-location `looks_generated`; consider ranking
   demotion.*
4. **Block locations carry no enclosing unit name** (`name: null` on every sampled
   block) — an agent cannot even name the region it is discussing. *Fix: extend the
   fragments' `enclosing_unit` recovery to all Block locations.*
5. **Rust inline `mod tests` is scope-tagged `prod`** — a one-line `is_test_loc` gap
   with outsized effect on triage. *(Filed separately as a bug.)*

## What this does NOT conclude

The sample is 18 families from 14 repos — a usability probe, not a benchmark. It says
nothing about ranking quality (the v5 labelset owns that), and "decidable from paths"
findings lean on agent-side world knowledge (locale layouts, fixture conventions) that
a stricter contract should not assume.
